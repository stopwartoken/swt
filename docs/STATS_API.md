# SWT Transparency Stats API

Read-only JSON endpoint exposing aggregated on-chain data for SWT.  
Intended for explorers, dashboards, researchers and anyone who wants to verify
the state of the token, liquidity locks and humanitarian fee.

---

## Endpoint

- **URL:** `https://stopwartoken.org/api/stats`
- **Method:** `GET`
- **Content-Type:** `application/json; charset=utf-8`
- **Refresh interval:** ~5 minutes
- **Network:** Polygon (chain id 137)

The endpoint is generated by an off-chain script that reads data directly from:

- SWT token contract  
- Uniswap v3 pool  
- QuickSwap Algebra pool  
- LP NFT locker contracts

No private keys are exposed or required to consume this endpoint.

## Field-by-Field Description

### **meta**
Basic static and indexing information.

- **token** – human-readable token name (`"STOPWARTOKEN"`)
- **symbol** – token symbol (`"SWT"`)
- **network** – network name (`"Polygon"`)
- **contract** – SWT contract address
- **decimals** – SWT decimals (`18`)
- **deployment_block** – block number where SWT was deployed
- **last_block** – latest processed block at the time of generation
- **last_update_utc** – ISO-8601 timestamp when this JSON was generated (UTC)

---

### **market**
Indicative price information.

- **price_usd** – estimated SWT price in USDT, derived from the main Uniswap v3 sqrtPriceX96  
- **price_source** – source identifier (currently `"UniswapV3"`)
- **pool_address** – Uniswap v3 SWT/USDT pool contract used for price calculation

> This section is informational only and should not be treated as a price oracle.

---

### **charity.current**
Current state of the humanitarian fee accumulated on the SWT contract.

- **accumulated_fee_swt** – SWT balance on the SWT contract that comes from the 0.1% fee  
- **accumulated_fee_usdt** – USDT currently held on the SWT contract (after `triggerCharity()` calls)  
- **charity_threshold_swt** – SWT threshold after which `triggerCharity()` becomes meaningful  
- **charity_threshold_reached** – boolean: `true` if the SWT threshold has been reached  
- **bounty_swt** – indicative SWT amount that may be used as a bounty for the caller of `triggerCharity()` once the threshold is reached

> The `bounty_swt` field is informational only and does **not** constitute a promise or guarantee.  
> Actual behavior is fully defined by the deployed smart contracts.

---

### **charity.lifetime**
Aggregated totals since deployment (based on on-chain data).

- **total_swt_collected** – total SWT amount ever collected as 0.1% fee  
- **total_usdt_collected** – total USDT ever obtained from converting collected SWT  
- **total_usdt_sent** – total USDT ever sent from the SWT contract to humanitarian recipients (via `charityTransfer` or equivalents)

These values are derived from on-chain balances and transfer logs and are provided for transparency and independent verification.

---

### **liquidity.summary**
High-level state of liquidity locks.

- **locked** – `true` if all tracked LP positions across all pools are currently locked
- **lp_unlock_date_utc** – earliest known unlock date among all *current* liquidity positions

If different positions have different unlock dates, the **earliest date** is reported.

---

### **liquidity.pools.uniswap_v3**  
### **liquidity.pools.quickswap**

Information about each DEX where SWT liquidity is deployed and locked.

Common fields:

- **pool_address** – on-chain address of the liquidity pool
- **sqrtPriceX96** – raw current pool price state in Q64.96 format
- **price_usd** – current SWT price in USDT/USD terms derived from this pool
- **name** – human-readable pool name (e.g., `"Uniswap V3"`, `"QuickSwap Algebra"`)
- **dex** – short identifier (`"UniswapV3"`, `"QuickSwap"`)
- **liquidity_usdt** – approximate total active liquidity value for this DEX expressed in USDT (derived from current pool state and tracked LP positions)
- **locker** – address of the LP-NFT locker contract used for this DEX  
- **nft_contract** – address of the LP-NFT contract (Uniswap v3 Positions NFT / Algebra NFT)
- **positions** – an array of tracked LP positions

> `sqrtPriceX96` is the raw pool price value returned by the AMM contract.
> It is included for transparency and independent verification.
>
> `price_usd` is the current SWT price derived from that pool state.
>
> `liquidity_usdt` is an approximate real-time valuation based on the current pool state.
> It reflects the USDT-equivalent liquidity deployed across tracked LP positions.
> The value may fluctuate as price moves across ranges.

Each `positions[]` element:

- **token_id** – LP-NFT tokenId  
- **liquidity_uint128** – raw liquidity value stored in the LP position
- **tickLower** – raw lower tick boundary of the LP position
- **tickUpper** – raw upper tick boundary of the LP position
- **range_min** – lower bound of the price range (SWT price in USDT, approximate)  
- **range_max** – upper bound of the price range  
- **current** – `true` for the position that currently holds active liquidity near market price  
- **unlock_date_utc**
  - ISO-8601 timestamp for positions that are already locked and have been scanned
  - `"locked_not_scanned"` for positions that are already locked on-chain but not yet scanned by the monitoring script

### **Notes**
- The endpoint is best-effort and may occasionally lag behind the latest block.  
- Consumers should implement caching, retries, and proper error handling.  
- Nothing here or in the JSON payload constitutes financial advice, investment solicitation, or guarantees.  
- This documentation is provided solely to support independent verification of SWT token mechanics and liquidity locks.
